Learn by doing, with your tools

Teach Me: your AI learning companion

As you code, debug, configure, analyze data, write documents, or research — Teach Me spots the moments worth learning from and turns key concepts, hidden mechanisms, and decision rationales into a local knowledge base. It also builds a growing learner portrait so the AI can teach you the way you like.

Install it for my agent See how it works

Works with Claude Code, Codex, OpenClaw, and Kimi Code CLI. Your data lives locally in ~/.teach_me_skill/vault. Git sync is optional and never enabled by default.

Why

Every finished task should leave you with something that is truly yours.

Teach Me automatically captures what is worth keeping from your work: concepts you understood, pitfalls you hit, tradeoffs you made, and risks you noticed. It moves you from “it works” to “I actually learned it.”

Learn from real work

No extra tutorials needed. The project, dataset, or configuration you are working on right now is the curriculum. Every turn can become a concept note or knowledge-tree node.

Hook-aware, not keyword-driven

Prompt, tool, and stop hooks observe actual behavior: edits, tests, builds, errors, verification, configuration changes, data processing. Notes are not triggered only when you say “teach me.”

Only the high-value points

By default it keeps just 1-3 transferable items: hidden mechanisms, decision rationales, reusable reasoning, future risks. No note flooding.

It draws your learner portrait

It tracks what you know, where you are fuzzy, and how you like to be taught. Explanations and reviews become increasingly personal instead of one-size-fits-all.

Shape the AI's teaching personality

Set a speaking style and teaching persona: strict mentor, curious peer, concise coach… Teach Me explains and probes in that voice.

Remember what you learn

Teach Me Recap uses spaced repetition and active recall to turn “I read it once” into “I can use it.”

Local-first and fully controllable

All readable notes are plain Markdown stored locally. Git sync is optional and disabled by default.

How

Four steps to turn a work phase into knowledge.

Teach Me does not replace your agent. It acts like a tutor who knows when to pause: it looks at the evidence first, then decides whether to summarize.

1. Sense the sceneWork, learning, debugging, research, or prompts like “teach me” / “why” inject Teach Me context.
2. Collect evidencePre / Post / Failure tool hooks log real edits, tests, builds, errors, verification, configuration changes, and data processing.
3. Score at StopAt the phase boundary, evidence is scored: weak signal is skipped, strong signal triggers one short review.
4. Capture assetsWrite only 1-3 high-value items: concepts, reusable reasoning, workflow maps, future risks. Update the knowledge tree, learner portrait, and review schedule.

Skills

Three skills for the full loop: capture, check, review.

Not one bloated tool, but three small skills that work together.

🌱 Teach Me

Core skill. Collects evidence while you work, triggers reviews at phase boundaries, writes high-value knowledge to the local vault, and updates your learner portrait.

🔍 Teach Me Check

Diagnostic skill. Checks installation status, vault location, learner portrait, users, and style, then suggests next steps in natural language.

🔄 Teach Me Recap

Review skill. Uses spaced repetition and active recall to help you review captured concepts so you do not forget them.

Hooks

Installation should be reversible, repeatable, and respectful of existing config.

Installers append Teach Me's own marker instead of replacing user config. Codex also enables [features] and adds the writable root once. Kimi rewrites only the hooks = [...] block.

PlatformRegistrationEventsTeach Me behavior
Claude Code ~/.claude/settings.json UserPromptSubmit, PreToolUse, PostToolUse, Stop Injects context for explicit learning or work prompts, scores tool evidence, and may request one short review at Stop.
Codex ~/.codex/config.toml UserPromptSubmit, PreToolUse, PostToolUse, Stop Enables [features] hooks = true, appends Teach Me hook blocks, and adds ~/.teach_me_skill as a writable root.
Kimi Code CLI ~/.kimi/config.toml UserPromptSubmit, tool events, Stop Reuses the shared ~/.agents/skills/teach-me path, records evidence, and triggers reviews through supported hook events.
OpenClaw ~/.openclaw/hooks/teach-me-learning message:received, agent:bootstrap Records whether the session is development-learning work, then appends Teach Me context to bootstrap files. True final-review behavior needs plugin support.

Hooks do not teach directly. They provide context and evidence. The agent still decides at the phase boundary whether anything deserves a durable note.

Install

Install the skill, then register hooks for the agent you use.

For the easiest path, paste the first prompt into your agent. Manual install is just as direct: install.sh copies the skill, and each platform's install-hook.sh connects hooks.

Ask an agent to install it skill + hooks

Send this to Claude Code, Codex, OpenClaw, or Kimi Code CLI.

Please install the Teach Me Agent Skill (this installs Teach Me, Teach Me Check, and Teach Me Recap):
1. Clone https://github.com/dull-bird/teach-me-skill.git
2. Enter teach-me-skill and run ./install.sh (only copy skill files; do not install extra dependencies or test tools)
3. If I use Claude Code, run ./claude-code/install-hook.sh
4. If I use Codex, run ./codex/install-hook.sh
5. If I use OpenClaw, run ./openclaw/install-hook.sh
6. If I use Kimi Code CLI, run ./kimi/install-hook.sh
7. After installation, run:
   python3 ~/.codex/skills/teach-me/scripts/teach_me.py context
   or the scripts/teach_me.py context command under the matching agent skills directory
8. Later, check status any time with:
   python3 ~/.codex/skills/check/scripts/check_me.py report
9. To review captured knowledge, say "Help me review" and the agent will run:
   python3 ~/.codex/skills/recap/scripts/recap.py next
10. Before writing the first learning note, ask me to confirm the vault path, note language, and whether to enable Git sync
