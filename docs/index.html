Agent Skill for development learning

Teach Me

把真实开发过程，沉淀成可复习的 Obsidian 知识笔记。它不打断实现，只在有价值的阶段边界捕获概念、算法思路、架构图谱和复盘问题。

安装 Teach Me 查看 hook 机制

适配 Claude Code、Codex、OpenClaw、Kimi Code CLI。默认 vault 路径是 ~/.teach_me_skill/vault。

$ python3 skills/teach-me/scripts/teach_me.py context
Teach Me learning context:
- vault: ~/.teach_me_skill/vault
- capture only high-value development knowledge
- keep implementation flow uninterrupted

What it does

不是把每个命令都讲一遍，而是留下以后真会用的知识。

Teach Me 把 agent 的开发上下文和一个本地学习运行时连起来。它的目标不是制造更多解释，而是把值得复用的工程判断写成稳定笔记。

少打扰

实现时正常推进。只有用户明确要求教学，或一个开发阶段自然结束时，才判断是否捕获学习笔记。

抓高价值

内置价值 rubric：新颖性、可迁移性、项目相关性、隐藏复杂度、未来 bug 风险、算法价值和用户信号。

能复习

输出到 Obsidian：概念、算法思路、项目地图、wikilinks、知识图谱、复习间隔和 Socratic questions。

Workflow

Hook 负责轻量上下文，Skill 负责教学判断，Runtime 负责落盘。

这种分层让行为更可控：hook 不替模型做学习决策，只告诉 agent 什么时候应该使用 Teach Me，以及本地 runtime 在哪里。

1. 识别开发类提示词或手动触发词进入 hook。
2. 注入向 agent 提供紧凑的 Teach Me context。
3. 判断阶段结束时按 rubric 选 1 到 3 个高价值知识点。
4. 沉淀runtime 写入 Obsidian vault 和学习状态。

Hooks

Hook 安装方式保持幂等，只注册 Teach Me 自己需要的入口。

重点是只追加自己的 marker、不覆盖用户原配置。Codex 会修补已有 [features]，Kimi 会用括号匹配重写 hooks = [...] 数组，避免破坏原有 hook。

平台注册位置事件Teach Me 行为
Claude Code ~/.claude/settings.json UserPromptSubmit、PreToolUse with Bash matcher 开发类提示词注入上下文；Bash 开发命令写入轻量事件日志。
Codex ~/.codex/config.toml [[hooks.UserPromptSubmit]]、[[hooks.PreToolUse]] 自动打开 [features] hooks = true，然后追加 Teach Me hook block。
Kimi Code CLI ~/.kimi/config.toml hooks = [{ event = "UserPromptSubmit", ... }] 复用共享 ~/.agents/skills/teach-me 路径，插入单条 UserPromptSubmit hook。
OpenClaw ~/.openclaw/hooks/teach-me-learning message:received、agent:bootstrap 先记录 session 是否是开发学习场景，再在 bootstrap 文件里追加 Teach Me context。

当前仓库的 hook 更保守：它只注入学习上下文和记录工具事件，真正要不要写学习笔记仍由 agent 在阶段边界按 skill rubric 判断。

Install

安装技能，再按你使用的 agent 注册 hook。

如果你让 agent 自动安装，直接复制第一段提示词。手动安装时，先运行根目录 install.sh，再按需要运行各平台的 install-hook.sh。

让 agent 自动安装 技能 + hook

把这段发给 Claude Code、Codex、OpenClaw 或 Kimi Code CLI。

请帮我安装 Teach Me 这个 Agent Skill：
1. 克隆 https://github.com/dull-bird/teach-me-skill.git
2. 进入 teach-me-skill，运行 ./install.sh
3. 如果我使用 Claude Code，运行 ./claude-code/install-hook.sh
4. 如果我使用 Codex，运行 ./codex/install-hook.sh
5. 如果我使用 OpenClaw，运行 ./openclaw/install-hook.sh
6. 如果我使用 Kimi Code CLI，运行 ./kimi/install-hook.sh
7. 安装后运行：
   python3 ~/.codex/skills/teach-me/scripts/teach_me.py context
   或对应 agent skills 目录下的 scripts/teach_me.py context
8. 第一次写学习笔记前，先让我确认 vault 路径和笔记语言

手动安装 本地 clone

git clone https://github.com/dull-bird/teach-me-skill.git
cd teach-me-skill
./install.sh

# Optional hooks
./claude-code/install-hook.sh
./codex/install-hook.sh
./openclaw/install-hook.sh
./kimi/install-hook.sh

第一次配置 Obsidian vault

默认 vault 是 ~/.teach_me_skill/vault，语言用 auto 时会跟随对话语言，常见技术名词保留英文。

python3 ~/.codex/skills/teach-me/scripts/teach_me.py configure --language auto

# Use a custom vault path
python3 ~/.codex/skills/teach-me/scripts/teach_me.py configure \
  --vault ~/Documents/Teach-Me-Vault \
  --language auto

Vault shape

写出来的是普通 Markdown，不被任何平台锁住。

Teach Me 的 runtime 会维护一个 Obsidian 兼容目录：可读笔记放在可见文件夹，机器状态放在 .teach-me/。

vault/
├── 00_Index.md
├── 01_Knowledge_Graph.md
├── 02_Concepts/
├── 03_Algorithmic_Ideas/
├── 04_Project_Maps/
├── 05_Socratic_Questions/
├── 06_Reviews/
└── .teach-me/
    ├── learning-state.json
    ├── style-profile.json
    └── events.jsonl
